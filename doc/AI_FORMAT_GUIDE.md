# AI 3D 模型數據格式指南

本文檔提供了 AI 生成 3D 模型數據的標準格式說明，用於確保生成的 JSON 數據能被正確渲染。

## 基本結構

數據應該是一個陣列，每個元素代表一個 3D 物件：

```json
[
  { "type": "floor", ... },
  { "type": "wall", ... },
  { "type": "column", ... },
  ...
]
```

## 支援的元素類型與屬性

### 1. 地板 (floor)

```json
{
  "type": "floor",
  "width": 10,    // 寬度，必須 > 0
  "height": 8     // 長度，必須 > 0
}
```

### 2. 牆壁 (wall)

```json
{
  "type": "wall",
  "x": 0,         // X軸位置
  "y": 1.25,      // Y軸位置，通常為高度的一半
  "z": 4,         // Z軸位置
  "width": 10,    // 寬度，必須 > 0
  "height": 2.5,  // 高度，必須 > 0
  "depth": 0.2    // 厚度，可選，默認為 0.1
}
```

### 3. 圓柱 (column)

```json
{
  "type": "column",
  "x": 3,         // X軸位置
  "y": 1.5,       // Y軸位置，通常為高度的一半
  "z": -2,        // Z軸位置
  "radius": 0.3,  // 半徑，必須 > 0
  "height": 3     // 高度，必須 > 0
}
```

### 4. 方柱 (column)

```json
{
  "type": "column",
  "shape": "square", // 指定為方形柱子
  "x": -3,           // X軸位置
  "y": 1.5,          // Y軸位置，通常為高度的一半
  "z": -2,           // Z軸位置
  "width": 0.5,      // 寬度，必須 > 0
  "depth": 0.5,      // 深度，可選，如果未提供則使用 width
  "height": 3        // 高度，必須 > 0
}
```

### 5. 門 (door)

```json
{
  "type": "door",
  "x": 0,         // X軸位置
  "y": 1,         // Y軸位置，通常為高度的一半
  "z": -4,        // Z軸位置
  "width": 1.0,   // 寬度，必須 > 0
  "height": 2.0,  // 高度，必須 > 0
  "depth": 0.1    // 厚度，可選，默認為 0.05
}
```

### 6. 窗戶 (window)

```json
{
  "type": "window",
  "x": 2,         // X軸位置
  "y": 1.5,       // Y軸位置，通常為高度的一半
  "z": 4,         // Z軸位置
  "width": 1.5,   // 寬度，必須 > 0
  "height": 1.0,  // 高度，必須 > 0
  "depth": 0.05   // 厚度，可選，默認為 0.05
}
```

## 注意事項

1. 所有數值必須為數字，不能是字符串
2. 維度值（width、height、depth、radius）必須大於0
3. 當指定 `"type": "column"` 時：
   - 如需圓柱，提供 `radius` 屬性
   - 如需方柱，設置 `shape: "square"` 並提供 `width` 和可選的 `depth` 屬性
4. 坐標系統：
   - X軸：水平（左右）
   - Y軸：垂直（上下）
   - Z軸：縱深（前後）
   - 原點(0,0,0)通常位於場景中心或地板中心
